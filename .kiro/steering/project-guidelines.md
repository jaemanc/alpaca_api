# Project Guidelines - Alpaca Real-time Alert System

## 아키텍처 원칙
- Kafka + Redis 기반 실시간 스트리밍 파이프라인 구성
- 최소한의 리소스 사용 (경량 컨테이너, 메모리 효율적 설계)
- 알림 채널: iOS 푸시 알림 우선, 불가 시 이메일 폴백

## 코드 작성 규칙
- 모든 코드에 상세한 한글 주석 작성
- 함수/클래스/모듈 단위로 목적과 동작 방식 설명
- 복잡한 로직에는 단계별 주석 포함

## 문서화
- README.md에 전체 시스템 스펙과 아키텍처 도식(Mermaid) 포함
- 각 컴포넌트의 역할과 데이터 흐름 명시

## 테스트 정책
- 검증을 위한 다수의 단위 테스트 파일 생성 금지
- testing-guidelines.md의 원칙을 준수
- wave 단위 통합 검증만 수행

## 리소스 최적화
- 불필요한 폴링 금지, WebSocket/SSE 기반 실시간 수신
- Redis는 캐시 + 상태 저장 용도로만 사용
- Kafka 파티션은 최소 구성 (필요 시 확장)
- 컨테이너 메모리 제한 설정 필수
